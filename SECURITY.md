# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub's private vulnerability reporting for this repository when it is
enabled. Until then, contact the project maintainer privately through the
contact information on the repository owner's GitHub profile and include:

- a description of the vulnerability
- affected versions or commits
- clear reproduction steps
- any suggested mitigation

Please avoid including real OAuth credentials, refresh tokens, combat logs, or
other personal data in a report.
